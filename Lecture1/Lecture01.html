Python vs C++:¶

Interpretability vs Compilability¶

  • In compiled languages, the program code must be compiled into the executable code. This code will then execute something that knows nothing about the original programming language.

  • In the interpreted language, the executor is the program itself, which analyzes the source code. Python is an interpreted language (and the executor of the code is a program called "python"). Several important properties follow from this: - Most errors occur at the program execution stage - More flexible and dense code

Strong vs dynamic typing¶

  • Strong: Each object has one specific type; it doesnt happen that we dont know what type of object it is, or that the object belongs to some undefined type that will be resolved later, or that we have objects of a certain universal type.
  • Dynamic: We almost cannot in any way understand from the source code of the program what type of object is in this expression. At least we cannot understand exactly. To find out, you need to interpret the entire program from the very beginning to this expression.

Memory management¶

No need to monitor allocated memory and clean it up: objects are deleted when there are no links to them. However, memory can still be overloaded.

For details see https://www.geeksforgeeks.org/garbage-collection-python/

            #  60 = 0011 1100 
            #  13 = 0000 1101

# x & y  Does a "bitwise and". 
# Each bit of the output is 1 if the corresponding bit of x AND of y is 1, otherwise it's 0.
60 & 13     #  12 = 0000 1100

# x | y  Does a "bitwise or". 
# Each bit of the output is 0 if the corresponding bit of x AND of y is 0, otherwise it's 1.
60 | 13     #  61 = 0011 1101 

# x ^ y  Does a "bitwise exclusive or". 
# Each bit of the output is the same as the corresponding bit in x if that bit in y is 0, 
# and it's the complement of the bit in x if that bit in y is 1.
60 ^ 13     #  49 = 0011 0001

# ~ x    Returns the complement of x - the number you get by switching each 1 for a 0 and each 0 for a 1. 
# This is the same as -x - 1.
~60         # -61 = 1100 0011

# x << y Returns x with the bits shifted to the left by y places (and new bits on the right-hand-side are zeros).
# This is the same as multiplying x by 2**y.
60 << 2     # 240 = 1111 0000 


# x >> y Returns x with the bits shifted to the right by y places. This is the same as //'ing x by 2**y.
60 >> 2     #  15 = 0000 1111
